# 参加者用ダッシュボード（マイページ）の実装計画

参加者がこれまでにフィードバックしたポスターと、その発表者情報（ポスタータイトル、発表者名、所属、メールアドレス）を一覧で確認できる「参加者用ダッシュボード（マイページ）」を実装します。

---

## 🎨 デザインコンセプト
- **プレミアム・グラスモルフィズム**: アプリ全体のトンマナに合わせ、美しい青のグラデーション背景、ゆっくりと動く光の球体、すりガラス調のカード（`glass-panel`）を全面的に採用します。
- **高い情報視認性**: ポスター番号、タイトル、発表者情報、自分が送信したフィードバック内容（興味レベル・コメント）を美しく整理してグループ化します。
- **モバイルフレンドリー**: 指先サイズのボタン、程よい余白、タップフィードバックを適用します。

---

## 🗄 データベース設計の拡張

現在、データベースには `interests` と `participants` テーブルのみが存在し、ポスター情報（タイトルや誰が発表しているか）が存在しません。
ポスタータイトルと発表者の情報をリレーショナルに管理するため、新たに `posters` テーブルを追加し、`interests` からの外部キー制約を設定します。

### 1. 新規作成: `posters` テーブル
ポスター番号（ID）、ポスタータイトル、および発表者ID（`participants` のIDを参照）を保持します。

```sql
-- ポスターテーブルの作成
CREATE TABLE public.posters (
  id integer PRIMARY KEY,
  title text NOT NULL,
  presenter_id text REFERENCES public.participants(id) ON DELETE CASCADE
);

-- RLS（Row Level Security）の無効化（匿名アクセス用）
ALTER TABLE public.posters DISABLE ROW LEVEL SECURITY;
```

### 2. `interests` テーブルの更新
`poster_id` カラムが `posters.id` を参照するように外部キー制約を追加します。

```sql
ALTER TABLE public.interests
ADD CONSTRAINT fk_interests_poster
FOREIGN KEY (poster_id) REFERENCES public.posters(id) ON DELETE CASCADE;
```

> [!NOTE]
> **安全な移行とシーディングについて**
> 既存 of `interests` データ（例: `poster_id: 33`）があるため、外部キー制約を追加する前に、対応するポスターデータを `posters` テーブルに挿入する必要があります。
> テストおよび動作確認がスムーズに行えるよう、**SQLスクリプトの提供**に加え、**自動でダミーデータを挿入するNode.js用のシードスクリプト**を作成して提供します。

## Proposed Changes

### 1. データベース構築・データ投入
- **`[NEW] scripts/seed.js`**:
  - `posters` テーブルを作成し、いくつかのサンプル発表者（Presenter）とポスタータイトル（例: No.1 〜 50）を自動でSupabaseに挿入するNode.jsスクリプト。
  - ユーザー様がSupabaseのSQLエディタで直接実行できるよう、SQLクエリもドキュメントとして提供します。

### 2. 参加者用ダッシュボード画面
- **`[NEW] app/my-dashboard/page.tsx`**:
  - ログイン中の参加者（`localStorage` の `userId`）のフィードバック履歴を表示するページ。
  - **未ログイン時**: 「参加者登録が必要です」という高級感あるグラスモルフィズムカードを表示し、`/register?redirect=/my-dashboard` への案内ボタンを設置。
  - **ログイン時**:
    - Supabaseから `interests` を `poster:posters(presenter:participants)` のリレーションで一括取得。
    - 各ポスター情報、発表者名、勤務先、所属、およびメールアドレスを表示。
    - 自分が登録した「興味レベル」「コメント」「連絡先共有のステータス」を表示。
    - 「このポスターのフィードバックを修正する」ボタンを設置し、`/poster/[posterId]` にワンタップで戻れるUXを提供。

### 3. ナビゲーションの動線改善
- **`[MODIFY] app/page.tsx`**:
  - すでに登録済みのユーザー（`userId` が `localStorage` にある）の場合、トップページに「マイページ（フィードバック一覧）」ボタンを表示するように変更。
- **`[MODIFY] app/poster/[posterId]/page.tsx`**:
  - ヘッダー部分に「マイページ」へのリンクボタンを美しく配置し、他のポスターを巡りながらいつでも自分の履歴に戻れるようにします。

## Verification Plan

### 1. データベースのセットアップとシーディング
- シードスクリプトを実行し、`posters` テーブルとサンプルデータがエラーなく作成されることを確認します。

### 2. ブラウザによる機能・表示検証（ブラウザサブエージェント）
- **未登録状態でのアクセス**:
  - `/my-dashboard` にアクセスした際、未登録の警告と「登録する」ボタンが美しく表示されるか確認。
- **登録後の表示**:
  - `/register` でID `123-456`（山田太郎様）などで登録完了後、`/my-dashboard` に遷移して「ようこそ、山田 太郎 様」と表示されるか確認。
  - ポスターへの興味をいくつか登録した後、ダッシュボードにそれらのフィードバックがリアルタイムでリスト化されるか確認。
  - 表示項目（ポスターNo、タイトル、発表者名、勤務先、所属、メールアドレス、コメント、共有ステータス）がプレミアムなグラスモルフィズムデザインで破綻なく表示されているか検証。
