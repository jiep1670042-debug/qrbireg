# 計画書：ポスターQRコード印刷機能のカスタマイズと個別印刷対応

イベント主催者がポスター掲示用のQRコードシートを印刷する際、会場のスペースや掲示方法に合わせてレイアウト（1枚あたり1個、2個、4個、6個配置）を選択でき、さらに特定のポスターのみを指定して個別印刷できるようにする機能を実装します。

---

## 💡 アーキテクチャ設計とカスタマイズ機能

### 1. 🖨️ 多彩な印刷レイアウト（面付け）の動的切り替え
- 管理画面の「QRコード印刷シート」タブに、レイアウト選択UI（ラジオボタンまたはカード型選択UI）を追加します。
  - **A4縦・1個配置**: 1ページに1枚の大きな案内カード（ポスター横のメイン掲示用など）。
  - **A4横・2個配置**: 横向き用紙に2個並べる（中サイズ）。
  - **A4縦・4個配置**: 縦向き用紙に2x2の4個配置。
  - **A4縦・6個配置**: 縦向き用紙に2x3の6個配置（受付配布用、または縮小掲示用）。
- 選択されたレイアウトに応じて、CSSの `@media print` メディアクエリ内の `@page { size: ... }` や改ページコンテナのグリッド幅、カード内の余白、文字サイズ、QRコード画像の大きさをJavaScript/Tailwindで動的に切り替えます。
- 印刷時にページ崩れを起こさないよう、指定面付け数（2, 4, 6個）ごとにデータをグループ化し、各グループに `page-break-after: always`（改ページ）を設定した印刷専用コンテナを生成します。

### 2. ☑️ 印刷対象ポスターの選択機能
- 「QRコード印刷シート」タブの上部に、印刷対象となるポスターを選択するためのリスト（チェックボックス付き）を設置します。
- 「すべて選択 / すべて解除」ボタンを配置し、ポスターNoやタイトルを確認しながら印刷したいものだけにチェックを入れて、一部のみを一括印刷できるようにします。
- 印刷プレビューおよび実際の印刷指示（`window.print()`）には、チェックされたポスターのみが抽出されて反映されます。

## 🧪 検証計画

### 手動検証
1. **印刷対象の個別選択**:
   - 印刷リストで特定のポスター（例: No.10 と No.30 のみ）にチェックを入れる。
   - 画面プレビューにその2枚のみが表示されるか確認。
   - 「印刷画面を開く」を押し、印刷ダイアログの送信プレビューに対象の2枚だけが含まれているか確認。
2. **1枚配置（A4縦）の検証**:
   - レイアウトで「1シートに1個 (A4縦)」を選択。
   - 印刷ダイアログで用紙向きが自動的に「縦（Portrait）」になり、1ページにカード1枚が画面いっぱいに中央配置されているか確認。
3. **2枚配置（A4横）の検証**:
   - レイアウトで「1シートに2個 (A4横)」を選択。
   - 印刷ダイアログで用紙向きが自動的に「横（Landscape）」になり、1ページに左右2列で綺麗に収まっているか確認。
4. **4枚配置（A4縦）の検証**:
   - レイアウトで「1シートに4個 (A4縦)」を選択。
   - 1ページに2x2のグリッドで4個配置され、自動的に次のページへ繰り越されるか確認。
5. **6枚配置（A4縦）の検証**:
   - レイアウトで「1シートに6個 (A4縦)」を選択。
   - 1ページに2x3のグリッドで6個配置されているか確認。
